# Korea Data API

Korean public holidays and data in English. Built for RapidAPI marketplace.

## Features

- Korean public holidays 2020-2030 with substitute holiday logic
- English + Korean names for all holidays
- Temporary holidays included (elections, special events)
- Type classification (fixed, lunar, substitute, temporary)

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | API info |
| GET | `/health` | Health check |
| GET | `/holidays/year/{year}` | All holidays for a year |
| GET | `/holidays/year/{year}/month/{month}` | Holidays in a month |
| GET | `/holidays/check/{date}` | Is this date a holiday? |
| GET | `/holidays/next` | Next upcoming holiday |
| GET | `/holidays/range?start=YYYY-MM-DD&end=YYYY-MM-DD` | Holidays in range |

## Run locally

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn app.main:app --reload
```

Visit http://localhost:8000/docs for interactive Swagger UI.

## Deploy

### Railway (recommended, free tier)

1. Push this repo to GitHub
2. Connect Railway to GitHub
3. Select this repo
4. Railway auto-detects Dockerfile in `deploy/Dockerfile`
5. Deploy - takes 2 min
6. Note the public URL (e.g., `https://korea-data-api.up.railway.app`)

### Render (alternative, free tier)

1. Create Web Service on Render
2. Connect GitHub repo
3. Build: `pip install -r requirements.txt`
4. Start: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

## RapidAPI submission

1. Go to https://rapidapi.com/provider → Add New API
2. Name: `Korea Data API`
3. Category: `Data`
4. Base URL: Railway/Render public URL
5. Import OpenAPI spec from `/openapi.json` (auto-generated by FastAPI)
6. Set pricing:
   - BASIC (Free): 100 requests/day
   - PRO ($5/mo): 10,000 requests/day
   - ULTRA ($20/mo): 100,000 requests/day
   - MEGA ($50/mo): unlimited
